2. Emmagatzematge local (local storage)

#800 #080 #008 #880 #808 #088 #888

Expliqueu de manera entenedora quins són els principals avantatges que ofereix l'emmagatzematge local d'informació.

Entre las principales ventajas de localStorage esta el ser una herramienta fundamental para optimizar el rendimiento, la funcionalidad y la seguridad de las aplicaciones web al almacenar datos directamente en el navegador del usuario teniendo en cuenta que se logra un acceso inmediato y fluido, incluso sin conexión a internet.

Esto se traduce en páginas web más optimas, con menor carga en el servidor y una experiencia de usuario personalizada que se mantiene entre sesiones. Además, localStorage ofrece un mayor control sobre la información privada, reduciendo el riesgo de filtraciones o accesos no autorizados.

Algunos ejemplos pueden ser

Ejemplos de uso:

  • Funcionalidad de favoritos: Muchos sitios web utilizan localStorage para permitir a los usuarios marcar y guardar contenido favorito directamente en su navegador. Esto garantiza que los usuarios puedan acceder fácilmente a su contenido preferido.

  • Información de inicio de sesión: Almacenar informacion de autenticación en localStorage permite a los usuarios mantenerse conectados automáticamente entre sesiones sin necesidad de volver a introducir sus datos cada vez que visitan el sitio.

  • Guardar el progreso (juegos, aplicaciones educativas, etc.): Aplicaciones como juegos y plataformas educativas pueden almacenar localmente el progreso del usuario, incluyendo niveles completados, logros desbloqueados o lecciones aprendidas. Esto permite una experiencia de usuario fluida y continua..

  • Aplicaciones de notas: Herramientas para tomar notas pueden utilizar localStorage para almacenar las notas del usuario localmente en su navegador. Esto asegura que los usuarios puedan acceder a sus notas rápidamente y editarlas.

Explique de forma muy clara y detallada (de nuevo, ELI5) qué sintaxis se debe seguir cuando queremos definir una clase

Para crear clases en JavaScript se usa la palabra clave class para definir la clase, seguido del nombre de la clase, despues, usando la palabra clave constructor para inicializar propiedades del objeto, al cual luego del las llaves se agrega la palabra especial this que se refiere al contexto actual, luego un punto para despues del mismo escribir el nombre de la propiedad,el siguiente paso seria agregar el operador de asignacion(=) y se finaliza la propiedad con el valor de la propiedad, que puede ser agregado como parametro y enviado como argumento al crear el objeto con la clase construida, despues de crear todas las propiedades, se le puede asinar un metodo que es comparable a una funcion que puede ser llamada por el objeto creado, esa funcion creada o metodo puede tanto retornar o no retornar valores y de ser necesario modificar el mismo objeto. 


      class NombreDeLaClase {
        constructor(parametros) {
          this.propiedad = valor;
        }
        metodo1() {
          // Código del método
        }
      }

Para crear un objeto a partir de la clase se usa la palabra clave new seguida del nombre de la clase y los argumentos requeridos por el constructor, del lado derecho de operador de asignacion , del lado izquiero se declara la variable la cual tendra todas las propiedades y metodos de la clase. 


      let nuevoObjeto = new NombreDeLaClase(argumentos);

Diferencia entre una clase y un objeto

  • Clase:Se puede definir como una plantilla que define las propiedades y metodos de un tipo de objeto. Es como un modelo general.
  • Objeto:Es una instancia de una clase, con valores específicos para sus propiedades y métodos.

...explicar com funciona a grans trets el procediment per escriure codi TypeScript en un projecte web.

Para escribir código TypeScript en un proyecto web, primero se debe instalar TypeScript globalmente y localmente en la carpeta del proyecto mediante npm (npm install -g typescript y npm install). Luego se verifica que este correctamente instalado escribiendo en la consola tsc -v, despues se abre la herramienta para escribir codigo que se use , se va directamente a la carpeta donde se instalo el copilador , se agrega un archivo tc y se inicia el copilador tsc indicando el documento a copilar, de esa forma se creara un archivo .js el cual sera leido por el navegador con el codigo copilado.

No es suficiente con cambiar la extensión de los archivos de .js a .ts. Los navegadores no pueden ejecutar archivos TypeScript directamente, por eso es necesario compilar los archivos TypeScript a JavaScript y en caso de que no de por alguna cuestion no funcione se puede resolver utilizando herramientas de construcción como webpack con ts-loader.
